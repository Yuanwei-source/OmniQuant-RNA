# OmniQuant-RNA 仓库协作约定

## 文档边界

远程仓库默认只保留三类正式文档：

1. 入口文档：`README.md`
2. 方法与边界文档：`docs/methods.md`
3. 输出与协作文档：`docs/outputs.md`、本文件

除非新增了长期稳定模块，否则不要继续新增同主题 Markdown。

## 本地材料与正式材料的区分

以下内容默认只保留本地，不作为远程仓库正式资产：

1. `experiments/audits/` 下的一次性排错或审计记录
2. `experiments/drafts/` 下的草稿
3. 临时 benchmark 方案
4. AI 生成但未采纳的长文说明

如果其中有长期有效结论，应提炼后并入正式文档，而不是原样推送。

## 修改文档的优先顺序

出现新需求时，优先顺序如下：

1. 先更新 `README.md`
2. 再更新 `docs/methods.md`
3. 再更新 `docs/outputs.md`

只有当现有文档实在承载不了时，才新增新的正式文档。

## 与 AI 协作时的约束

1. 优先改已有正式文档，不新建重复说明
2. 排错、脑暴、审计全文默认放本地，不进正式文档
3. 提交前确认新增 Markdown 是否真的会长期维护
4. 对外口径必须和代码默认行为一致，避免“文档一套、配置一套”

## 推荐的最小正式文档集合

当前仓库推荐长期保留的正式文档只有：

1. `README.md`
2. `docs/methods.md`
3. `docs/outputs.md`
4. `docs/repository-guidelines.md`

如果未来新增文档，应先说明它为什么不能被上述 4 份吸收。

## 删除原则

以下情况优先删除或转为本地资料：

1. 同一主题存在多份 benchmark 叙事版本
2. 文件名带明显阶段性痕迹，例如“审核结果_1”“审计结果_1”
3. 内容已经被正式文档吸收
4. 只对当时一次排错有价值
